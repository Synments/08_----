## 模型訓練與評估 (Model Training & Evaluation) 
### 資料集切分 (Train-Test Split)
- **操作**：將原始資料集隨機切分為 **訓練集 (80%)** 與 **測試集 (20%)**。    
- **分層抽樣 (Stratify)**：由於詐騙樣本極為稀少（僅約 4.8%），使用了 `stratify=y` 參數。這確保了訓練集與測試集中的詐騙比例完全一致，避免因隨機抽樣導致模型「沒考過詐騙題」或「考題分配不均」的問題。    
- **防止資料洩漏 (Data Leakage)**：嚴格遵守「先切分、再轉換」的原則。所有特徵工程 (例如 TF-IDF 統計) 皆在切分後才進行，確保測試集對模型而言是完全未知的環境。
### TF-IDF 向量化 (TF-IDF Vectorization) 

這是將文字轉化為數學語言的關鍵步驟：
- **TF-IDF 權重計算**：不單純統計字數，而是衡量詞彙的「辨識價值」。    
    - **TF (詞頻)**：某個詞在該職缺中出現次數越多，代表該詞對該職缺越重要。        
    - **IDF (逆向文件頻率)**：如果一個詞在所有職缺中都頻繁出現 (例如 "the", "work")，則調低其權重。        
- **結果**：這讓模型能自動鎖定具備高度辨識力的風險詞彙 (例如 "Cash", "Immediate", "No experience")，而不會被常見公文用語干擾。
### 自動化管線 (Scikit-Learn Pipeline) 

為了提升部署效率，將**特徵轉換**與**模型預測**封裝在 **Pipeline** 中：
- **流程**：`TF-IDF 向量化` $\rightarrow$ `隨機森林分類器`。  
- **優勢**：模型在預測新資料時，會自動套用與訓練時完全相同的清洗與轉換規則，避免「訓練與應用不一致 (Training-Serving Skew)」的風險。

---
### 演算法選擇：隨機森林 (Random Forest) — 建立基準

首先選擇 **隨機森林** 作為本專案的「基準模型 (Baseline)」，理由如下：
- **穩健性高 (Robustness)**：能有效處理文字特徵（高維度）與類別特徵（Logo/Questions）的混合數據。    
- **抗過擬合 (Overfitting)**：透過多棵決策樹的投票機制，在初始資料量較少時比單一模型更穩定。    
- **透明度**：其具備「特徵重要性」分析功能，能協助驗證我們先前的特徵工程方向是否正確。
### 評估指標與基準成績單 (Baseline Metrics) 

在 20% 的測試集上進行首輪驗證。雖然整體準確率 (Accuracy) 高達 97%，但針對 **「詐騙類別 (標籤 1)」** 的細部指標揭露了潛在風險：

| 指標 (Metric)    | 數值 (Value) | 專業解釋                                |
| -------------- | ---------- | ----------------------------------- |
| 準確率 (Accuracy) | 1.00       | 標記為詐騙的樣本中，確實為詐騙的比例。此階段達成 **「零誤判」**。 |
| 召回率 (Recall)   | 0.49       | **核心痛點**。僅成功抓到約一半的詐騙職缺，漏網之魚過多。      |
| F1-Score       | 0.66       | 精準度與召回率的調和平均，反映模型初始的綜合實力仍有大幅提升空間。   |
### 基準模型 - 混淆矩陣分析 (Confusion Matrix)

透過視覺化矩陣，觀察到初始 Baseline 的行為特徵：
- **高安全性，低偵測率**：模型表現極度保守。雖然完全沒有冤枉任何正常職缺 (False Positive = 0)，但卻放過了高達 **88 筆** 詐騙職缺。 
- **核心洞察**：模型對於「特徵不夠極端」的詐騙案例缺乏敏感度。這證實了單靠預設參數的隨機森林，無法應對真實環境中狡猾多變的詐騙文案。
### 模型優化：處理不平衡與增強特徵 (Model Optimization)

針對基準模型 Recall 過低 (漏抓率高) 的問題，進行了「特徵」與「演算法權重」的雙重優化：
- **深化文字特徵 (Advanced TF-IDF)**：    
    - **擴大詞彙量**：將 `max_features` 從 500 提升至 **2000**，以捕捉更細微的詐騙特徵詞。        
    - **導入雙字詞 (N-grams)**：設定 `ngram_range=(1, 2)`。這使模型具備辨識詞組的能力（例如 "work from home"、"money back"），而非僅能識別單個單字，有效提升了對詐騙語境的理解。        
- **類別權重平衡 (Class Weight Balancing)**：    
    - **操作**：在隨機森林中加入 `class_weight='balanced'`。        
    - **目的**：這如同給予模型一個「放大鏡」，強制模型在訓練時提高對少數族群（詐騙職缺）的關注度，對錯過詐騙樣本給予更高的懲罰。
### 優化版隨機森林成績單 (Optimized RF Results)

經過優化後，模型在測試集上的表現如下：

| 指標 (Metric)     | 數值 (Value) | 專業解釋                                     |
| --------------- | ---------- | ---------------------------------------- |
| 精確率 (Precision) | 1.00       | 依然維持極致的精準，標記為詐騙的樣本全部正確（零誤判）。             |
| 召回率 (Recall)    | 0.50       | **遭遇瓶頸**。儘管導入 N-grams 與權重調整，偵測率僅微幅提升 1%。 |
| F1-Score        | 0.67       | 綜合表現穩定，但對於防詐實務而言，「漏網之魚」依然過多。             |
### 核心洞察：演算法的侷限性 
- **頑固的漏網之魚**：即便特徵更豐富、權重更平衡，模型仍漏抓了 **86 筆** 詐騙職缺 (僅比基準版多抓到 1 筆)。    
- **結論**：隨機森林傾向於透過「多數決」產生預測，在處理極端不平衡的文字資料時，其決策邏輯天生過於「保守」。    

> **轉折點**：這次實驗讓我意識到，若要顯著提升偵測率 (Recall)，單靠特徵微調已進入邊際效益遞減階段。必須嘗試使用其他的演算法 —— **XGBoost** 與 **SVM**。

---
### 演算法：XGBoost (Extreme Gradient Boosting)

為了突破隨機森林的偵測瓶頸，導入了具備「梯度提升」機制的 **XGBoost**。其核心優勢在於：
- **序列學習 (Sequential Learning)**：模型會針對前一棵樹「抓不到」的詐騙樣本進行強化學習。    
- **不平衡加權 (scale_pos_weight)**：透過調整懲罰權重，強迫模型在損失函數中給予詐騙樣本更高的發言權。
### XGBoost - 實驗 A
- **設定**：將 `scale_pos_weight` 設為 **19.64** (負樣本/正樣本比例)，完全對齊資料分布。    
- **表現**：    
    - **Recall (召回率) 暴增至 0.79**：從原本只能抓到一半，提升到能識破近八成的詐騙。        
    - **Precision (精確率) 降至 0.59**：由於模型變得極度敏銳，產生了較多誤報（96 筆正常職缺被標記為詐騙）。        
- **結論**：此版本適合「寧可錯殺、不可放過」的極高安全性場景，但人工審核成本較高。

| 指標 (Metric)     | 標籤 1 (詐騙) 數值 | 專業解析                             |
| --------------- | ------------ | -------------------------------- |
| 精確率 (Precision) | 0.59         | 警報中約 40% 為誤報，審核負擔較重。             |
| 召回率 (Recall)    | 0.79         | **重大突破**。成功攔截近八成的詐騙職缺（僅漏抓 36 筆）。 |
| F1-Score        | 0.67         | 整體性能與優化後的隨機森林持平，但能量轉向偵測端。        |
### XGBoost - 實驗 A：混淆矩陣洞察

觀察混淆矩陣，真陽性 (TP) 達 **137 筆**。雖然誤抓了 96 筆正常職缺，但在極度要求安全的場景下，這是目前最強的防禦機制。
### XGBoost -  實驗 B

為了在「偵測率」與「審核成本」間取得平衡，進行了參數微調：
- **設定**：將 `scale_pos_weight` 調降至 **10**，讓模型在標記詐騙時變得更為「審慎」。    
- **最終成績單 (精準版)**：

| 指標 (Metric)     | 數值 (Value) | 專業解釋                            |
| --------------- | ---------- | ------------------------------- |
| 精確率 (Precision) | 0.75       | 警報準確度大幅回升，有效降低誤報干擾。             |
| 召回率 (Recall)    | 0.73       | 依然維持高偵測率，成功攔截絕大多數詐騙。            |
| F1-Score        | 0.74       | **全系列最高**。代表模型在精準與召回間達成了「黃金平衡」。 |
### XGBoost - 實驗 B：混淆矩陣洞察
- **成效**：誤報數從 96 筆減半至 **42 筆**，同時成功守住了 **127 筆** 詐騙偵測。  
- **決策意義**：這個版本的 F1-Score (0.74) 證明了在不平衡資料中，適度的權重懲罰（而非極端懲罰）能帶給模型最佳的綜合預測品質。

### 演算法：SVM (Support Vector Machine) 

為了驗證不同數學邏輯對文字特徵的敏感度，導入了在文字分類領域極具聲望的 **SVM**。
- **核心邏輯**：不同於樹狀模型的「是非題」切割，SVM 試圖在 2000 維的特徵空間中尋找一個「最佳超平面 (Hyperplane)」，將詐騙與正常職缺徹底分隔。    
- **參數設定**：使用 `kernel='linear'` (線性核) 以應對高維 TF-IDF 特徵，並開啟 `class_weight='balanced'` 以應對不平衡數據。

| 指標 (Metric)     | 標籤 1 (詐騙) 數值 | 專業解釋                             |
| --------------- | ------------ | -------------------------------- |
| 精確率 (Precision) | 0.50         | **主要缺點**。警報中僅一半為真，誤報率較高。         |
| 召回率 (Recall)    | 0.86         | **全場最高**。成功攔截 86% 的詐騙（僅漏抓 25 筆）。 |
| F1-Score        | 0.63         | 雖偵測力強，但因誤報過多，綜合得分低於 XGBoost。     |
### 混淆矩陣洞察 
* 觀察紅紫色矩陣（`image_b88b87.png`），真陽性 (TP) 達到驚人的 **148 筆**。這證明了 SVM 的超平面對於「邊界模糊」的詐騙特徵具有極高的敏感度。

### 業務邏輯

- **平臺管理者 (省人工)**：選擇 **XGBoost (Precision-Focused)**。它維持了 0.74 的平衡表現 (F1 Score)，同時誤判極少，節省大量人工審核成本。    
- **用戶安全官 (保安全)**：選擇 **SVM**。雖然會造成審核員的困擾，但它只放過了 25 個詐騙，安全性是最高的。
